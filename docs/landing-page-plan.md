# 🎯 유저 리서치 솔루션 선호도 조사 — 랜딩페이지 기획 문서

> **문서 버전**: v1.0  
> **작성일**: 2026-02-18  
> **목적**: 3가지 유저 리서치 솔루션 후보에 대한 잠재 고객의 실제 선호도(Skin-in-the-game 데이터)를 수집하는 랜딩페이지 기획

---

## 1. 실험 설계 (Pretotyping Framework)

### 1-1. 검증할 가설

| # | 가설 | 위험도 | 측정 지표 |
|---|------|--------|-----------|
| H1 | 초기 창업자/PM은 유저 리서치의 필요성을 인지하지만, "인터뷰이 모집"이 가장 큰 진입장벽이다 | 높음 | 페이지 내 공감 섹션 스크롤 깊이, 체류 시간 |
| H2 | 3가지 솔루션 중 하나가 명확히 선호된다 (→ 어떤 것인지는 데이터로 확인) | 핵심 | CTA 클릭 분포 (옵션별 클릭률) |
| H3 | "AI 합성 유저"보다 "실제 데이터 기반 AI"가 신뢰도가 높게 인식된다 | 중간 | 옵션1 vs 옵션2 클릭 비율 |
| H4 | 가격 민감도가 선호를 좌우한다 (저렴한 옵션에 쏠림 vs 품질 우선) | 중간 | 가격 정보 노출 전/후 선택 변화 |

### 1-2. 성공 기준 (Go / No-Go)

- **Go**: 7일 내 50명 이상 CTA 클릭, 특정 옵션에 40% 이상 집중
- **Pivot**: CTA 클릭은 있으나 옵션 간 균등 분포 (33:33:33) → 차별화 포인트 재설계
- **Kill**: 7일 내 CTA 클릭 20명 미만 → 타겟 또는 문제 정의 자체를 재검토

### 1-3. YODA 데이터 수집 설계

| 데이터 유형 | 수집 방법 | Skin-in-the-game 수준 |
|-------------|-----------|----------------------|
| 옵션 선택 클릭 | 랜딩페이지 CTA 버튼 | ★★☆ (행동 데이터) |
| 이메일 제출 | 얼리엑세스 신청 폼 | ★★★ (개인정보 제공 = 의지 표현) |
| 간단 설문 응답 | 선택 후 2문항 팝업 | ★★☆ (추가 시간 투자) |
| 가격 수용도 | 예상 가격 제시 후 "이 가격이면 쓰겠다" 체크 | ★★★ (결제 의사 표현) |

## 2. 타겟 오디언스

### 2-1. 핵심 타겟 (Primary)

**"유저 리서치가 필요한데 못 하고 있는 초기 창업자"**

- 예비창업자 / 초기 스타트업 창업자 (0→1 단계)
- 1인 메이커 / 솔로프리너
- 정부지원사업 준비 중인 예비창업자 (사업계획서에 고객 검증 필요)

### 2-2. 확장 타겟 (Secondary)

- 주니어 PM/PO (리서치 경험 부족, 예산 제한적)
- UX/UI 디자이너 (사용성 테스트 대상자 필요)
- 대학원생 (논문 리서치용 인터뷰이 필요)

### 2-3. 타겟의 핵심 고통 (참고자료 기반 분석)

참고자료에서 추출한 실제 고객의 언어:

> "인터뷰 중요한 건 알겠는데.. 인터뷰이 구하기 힘드시죠?" — 그로스헬퍼(디스퀴엇)
>
> "제 사업 고객이 좀 특수하여 데이터 모집이 쉽지 않았을 텐데" — 크몽 리뷰
>
> "인터뷰이를 구하는 것부터 막막하죠" — 그로스헬퍼 블로그
>
> "지인은 관계가 있는 사람이다보니 인터뷰 대답이 편향될 가능성" — 그로스헬퍼
>
> "커뮤니티에 모집글을 올리면 홍보글로 간주되어 삭제, 영구정지 위험" — 그로스헬퍼
>
> "DM이나 댓글은 회신이 잘 오지 않습니다" — 그로스헬퍼

**핵심 고통 3가지 (우선순위 순):**

1. **모집의 어려움**: 원하는 특성의 인터뷰이를 찾는 것 자체가 막막함
2. **비용 부담**: 기존 리서치 서비스는 FGI 6명에 300만원, 인당 최소 2만원+스크리닝 비용
3. **시간 소요**: 모집→스크리닝→일정조율→인터뷰→분석까지 최소 2~4주

## 3. 랜딩페이지 구조 (섹션별 상세 기획)

### 전체 플로우

```
[Hook 헤드라인] → [공감 섹션] → [기존 방법의 한계] → [3가지 솔루션 제시] → [비교표] → [CTA] → [얼리엑세스 폼] → [간단 설문]
```

---

### 섹션 1: Hook (히어로 섹션)

**목적**: 3초 안에 타겟의 고통을 자극하고, "여기 내 문제의 답이 있다"는 기대감 생성

**헤드라인 후보 (A/B 테스트 대상):**

- A안: **"인터뷰이 구하다 포기한 적 있으신가요?"**
- B안: **"유저 리서치, 중요한 건 아는데 — 시작이 안 되죠"**
- C안: **"고객 인터뷰 없이 만든 제품, 42%가 실패합니다"**

**서브 헤드라인:**
> 인터뷰이 모집부터 인사이트 도출까지, 당신의 상황에 맞는 유저 리서치 방법을 찾아보세요.

**CTA 버튼**: "내게 맞는 리서치 방법 찾기 ↓"  
*(스크롤 유도, 이탈 방지)*

---

### 섹션 2: 공감 (Pain Amplification)

**목적**: "이 사람들 내 상황을 정확히 아는구나" → 신뢰 형성

**구성**: 체크리스트 형태로 공감 유도

**카피:**

> **혹시 이런 경험 있으신가요?**
>
> ☐ 유저 인터뷰가 중요한 건 알지만, 인터뷰이를 어디서 구해야 할지 모르겠다  
> ☐ 지인에게 부탁하자니 솔직한 답변을 기대하기 어렵다  
> ☐ 커뮤니티에 모집글을 올렸다가 삭제당하거나 무시당했다  
> ☐ 리서치 업체에 맡기고 싶지만, 수백만 원의 비용이 부담된다  
> ☐ 결국 '내 감'으로 제품을 만들고, 출시 후에야 고객 반응을 확인한다  
> ☐ 정부지원사업 사업계획서에 '고객 검증' 항목을 어떻게 채울지 막막하다

**하단 멘트:**
> 2개 이상 체크하셨다면, 아래 솔루션이 도움이 될 수 있습니다.

---

### 섹션 3: 기존 방법의 한계 (Before → After 프레임)

**목적**: 현재 대안들의 문제점을 짚어 "새로운 솔루션이 필요하다"는 논리적 근거 제공

**구성**: 3칸 카드 레이아웃

| 방법 | 문제점 |
|------|--------|
| **지인 인터뷰** | 관계 때문에 솔직한 피드백 불가, 편향된 데이터 |
| **커뮤니티/SNS 모집** | 삭제 위험, 낮은 응답률, 스크리닝 불가 |
| **리서치 업체 의뢰** | FGI 6명에 300만원, 최소 2~4주 소요 |

**전환 멘트:**
> 그래서 저희는 **상황과 예산에 맞는 3가지 유저 리서치 방법**을 준비하고 있습니다.  
> 어떤 방식이 가장 끌리시나요?

### 섹션 4: 3가지 솔루션 (핵심 — 선호도 측정 구간)

**목적**: 3가지 옵션을 동등한 매력도로 제시하고, 실제 클릭 데이터로 선호도 측정

**레이아웃**: 3칸 카드 (모바일에서는 세로 스택), 각 카드에 개별 CTA 버튼

---

#### 옵션 A: AI 페르소나 인터뷰

**태그**: `⚡ 가장 빠름` `💰 가장 저렴`

**카드 타이틀**: **AI가 만든 가상 고객과 대화하기**

**설명 카피:**
> AI가 당신의 타겟 고객 페르소나를 생성합니다.  
> 성별, 나이, 직업, 라이프스타일, 구매 성향까지 설정하면  
> 즉시 인터뷰를 시작할 수 있습니다.
>
> 새벽 2시에도, 주말에도, 원하는 만큼 반복 가능.

**핵심 특징:**
- 🕐 소요 시간: 설정 5분, 인터뷰 즉시 시작
- 💰 예상 비용: 월 구독 또는 건당 소액
- 📊 결과물: AI 인터뷰 대화록 + 자동 인사이트 요약
- ⚠️ 참고: AI가 학습 데이터 기반으로 응답하므로, 완전히 새로운 시장에서는 실제 데이터와 차이가 있을 수 있음

**CTA 버튼**: `이 방식이 끌린다 →`

---

#### 옵션 B: 데이터 기반 AI 페르소나 인터뷰

**태그**: `🎯 가장 정확` `🧠 데이터 기반`

**카드 타이틀**: **실제 리서치 데이터로 훈련된 AI 고객과 대화하기**

**설명 카피:**
> 단순 AI가 아닙니다.  
> 실제 유저 인터뷰 수백 건의 데이터를 학습한 AI 페르소나입니다.  
> 진짜 고객이 했던 말투, 불만, 숨겨진 니즈까지 반영합니다.
>
> "그럴듯한 답변"이 아니라 "실제 고객이 했을 법한 답변"을 제공합니다.

**핵심 특징:**
- 🕐 소요 시간: 설정 10분, 인터뷰 즉시 시작
- 💰 예상 비용: 옵션 A보다 높지만, 실제 인터뷰 대비 1/5 이하
- 📊 결과물: 데이터 기반 인터뷰 대화록 + 근거 데이터 출처 표시 + 인사이트 리포트
- ✅ 차별점: Stanford 연구 기반 — 실제 사람의 재응답 일관성과 85% 수준의 정확도

**CTA 버튼**: `이 방식이 끌린다 →`

---

#### 옵션 C: 실제 사람 인터뷰 + 매칭

**태그**: `👥 가장 확실` `🤝 실제 고객의 목소리`

**카드 타이틀**: **진짜 고객을 만나서 직접 대화하기**

**설명 카피:**
> 유저 리서치의 정석.  
> 원하는 특성(나이, 직업, 관심사, 사용 앱 등)에 맞는  
> 실제 인터뷰이를 매칭해드립니다.
>
> 모집의 번거로움 없이, 준비된 사람과 바로 인터뷰하세요.

**핵심 특징:**
- 🕐 소요 시간: 매칭 1~3일, 인터뷰 30~60분
- 💰 예상 비용: 인당 매칭 비용 발생 (기존 업체 대비 저렴한 가격 목표)
- 📊 결과물: 인터뷰 녹음/전사 + AI 인사이트 분석 지원
- ✅ 차별점: 실제 Skin-in-the-game 데이터, 표정/뉘앙스/예상치 못한 발견

**CTA 버튼**: `이 방식이 끌린다 →`

### 섹션 5: 한눈에 비교하기

### 섹션 6: CTA + 얼리엑세스 폼

**목적**: 이메일 수집 = 가장 강력한 YODA 데이터

**구성:**

**헤드라인**: "가장 끌리는 방식을 선택하고, 얼리엑세스를 신청하세요"

**폼 구성:**
1. **선호 솔루션 선택** (필수, 라디오 버튼)
   - ⚡ AI 페르소나 인터뷰 (가장 빠르고 저렴하게 시작)
   - 🎯 데이터 기반 AI 페르소나 (정확도와 속도의 균형)
   - 👥 실제 사람 매칭 인터뷰 (가장 확실한 고객의 목소리)

2. **이메일 주소** (필수)

3. **현재 상황** (선택, 드롭다운)
   - 아이디어 단계 (아직 제품 없음)
   - MVP/프로토타입 있음
   - 이미 서비스 운영 중
   - 기타

4. **한 줄 고민** (선택, 텍스트)
   > "유저 리서치와 관련해서 가장 어려운 점 한 가지만 알려주세요"

**제출 버튼**: `얼리엑세스 신청하기`

**제출 후 화면:**
> 🎉 신청이 완료되었습니다!  
> OOO명이 함께 기다리고 있습니다.  
> 빠른 소식을 이메일로 전해드릴게요.

*(실시간 신청자 수 표시 = 사회적 증거 + 긴급성)*

### 섹션 7: 제출 후 미니 설문 (팝업 / 별도 페이지)

**목적**: 추가 정성 데이터 수집 (응답률 50% 이상 목표)

**2문항만:**

**Q1.** 선택한 옵션을 고른 가장 큰 이유는? (단일 선택)
- 가격이 가장 중요해서
- 속도가 가장 중요해서  
- 정확도/신뢰도가 가장 중요해서
- 실제 사람의 반응이 중요해서
- 기타 (자유 입력)

**Q2.** 유저 리서치에 월 얼마까지 쓸 수 있으신가요? (단일 선택)
- 0원 (무료만 사용)
- 1~5만원
- 5~15만원
- 15~30만원
- 30만원 이상

## 4. 카피라이팅 가이드

### 4-1. 톤 & 보이스

- **친근하지만 전문적**: 반말 ❌, 과도한 존댓말 ❌ → "~하세요" 체
- **공감 우선, 설득 나중**: 먼저 고통을 인정하고, 그 다음 솔루션 제시
- **과장 금지**: "혁명적", "최초의", "완벽한" 같은 수식어 자제
- **구체적 숫자 사용**: "빠르게" → "5분 안에", "저렴하게" → "인당 5천원부터"

### 4-2. 참고자료에서 빌려올 표현 (Voice of Customer)

| 출처 | 표현 | 활용 방안 |
|------|------|-----------|

## 5. 디자인 가이드

### 5-1. 레이아웃 원칙

- **모바일 퍼스트**: 타겟층(초기 창업자)은 모바일 비중 높음
- **1스크롤 = 1메시지**: 각 섹션이 한 화면에 하나의 메시지만 전달
- **3카드 동등 배치**: 솔루션 섹션에서 특정 옵션에 시각적 편향 주지 않음 (크기, 색상, 위치 모두 동일)
- **CTA 반복**: 솔루션 카드 개별 CTA + 비교표 하단 CTA + 폼 섹션 CTA

### 5-2. 편향 방지 설계

| 편향 유형 | 방지 방법 |
|-----------|-----------|
| 위치 편향 (좌→우 선호) | 모바일에서 카드 순서 랜덤화, 또는 A/B로 순서 변경 테스트 |
| 앵커링 효과 (첫 번째 옵션 선호) | 3가지 옵션의 순서를 방문자마다 변경 |
| 프레이밍 효과 | 각 옵션의 장점 수를 동일하게 맞춤 (각 4개씩) |
| 가격 편향 | 구체적 가격 대신 상대 비용(💰 개수)으로 먼저 표시 |

### 5-3. 색상 & 시각 요소

- 각 옵션별 구분 색상 (단, 동일 채도/명도로 선호도 편향 방지)
  - 옵션 A: 블루 계열 (기술/혁신 느낌)
  - 옵션 B: 퍼플 계열 (정확도/지능 느낌)
  - 옵션 C: 그린 계열 (신뢰/연결 느낌)
- 아이콘: 각 옵션별 직관적 아이콘 (로봇, 뇌+데이터, 사람 아이콘)

### 7-1. 핵심 지표 (Key Metrics)

| 지표 | 측정 도구 | 목표 |

| 스크롤 깊이 (솔루션 섹션 도달률) | Hotjar / GA 이벤트 | 60% 이상 |
| 옵션별 CTA 클릭 수 | GA 이벤트 (option_a_click 등) | 총 50건 이상 |
| 이메일 제출 수 (전환율) | 폼 제출 이벤트 | 방문 대비 10% 이상 |
| 미니 설문 응답 수 | 폼 제출 이벤트 | 이메일 제출자 대비 50% |

### 7-2. 분석 프레임워크

**1차 분석 (정량):**
- 옵션별 클릭 비율: A:B:C = ?:?:?
- 유입 채널별 선호 차이 (사푼사프 vs 디스퀴엇 vs 기타)
- 현재 상황(아이디어/MVP/운영 중)별 선호 차이

**2차 분석 (정성):**
- "한 줄 고민" 자유 응답에서 반복되는 키워드 클러스터링
- 선호 이유(Q1)와 WTP(Q2)의 교차 분석
- 예상과 다른 패턴 발견 시 → 추가 인터뷰 5명 진행

### 7-3. 의사결정 매트릭스

| 결과 시나리오 | 의미 | 다음 액션 |
|--------------|------|-----------|
| A(AI 페르소나)에 집중 | 속도와 가격이 최우선 | AI 페르소나 MVP 먼저 구축, 정확도는 후순위 |
| B(데이터 기반 AI)에 집중 | 정확도에 대한 니즈가 높음 | 데이터 파트너십/수집 전략 선행 |
| C(실제 사람 매칭)에 집중 | AI 불신, 확실한 결과 선호 | 매칭 플랫폼 MVP (그로스헬퍼 모델 참고) |
| A+B 합산 > 60% | AI 기반 접근이 시장 수요 | AI 솔루션 우선, C는 프리미엄 옵션으로 후행 |
| C > 50% | 전통적 리서치 수요가 여전히 강함 | 매칭 서비스 선행 + AI 분석 도구로 차별화 |
| 균등 분포 | 명확한 선호 없음 | 3가지 모두 가볍게 프리토타이핑 or 포지셔닝 재설계 |
